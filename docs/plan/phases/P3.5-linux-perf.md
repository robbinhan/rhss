# P3.5 — Linux 性能优化(目标:顺序 GB/s)

**目标**:把 Linux 路径推到顺序 1-3 GB/s,达到 D20"轻量服务器存储后端"档位;CI 加性能基准红灯保护(D21)。

**预计**:~ 2 周

**依赖**:P3 完成且合并。

> 仅对 Linux 编译路径启用,macOS 不变。所有改动用 `#[cfg(target_os = "linux")]` 隔开。详见 [architecture.md §4.10b](../architecture.md#410b-linux-性能优化路径目标顺序-gbs)。
>
> **macOS-only 用户**:可跳过此阶段,直接进 P4。

## 任务

| # | 任务 | 涉及文件 |

| 3.5.6 | Linux 性能基准 CI:`fio` 4K 随机读 + 1M 顺序读 + `dd` 10G 顺序写,阈值红灯 | `.github/workflows/perf.yml`, `tests/bench/` |

## 验收(Linux only,在 GitHub Actions Linux runner 上)

- [ ] `fio --rw=read --bs=1M --size=10G --iodepth=8` 吞吐 **> 1 GB/s**
- [ ] `fio --rw=randread --bs=4k --iodepth=32 --size=1G` IOPS **> 50K**
- [ ] `dd if=/dev/zero of=/mnt/rhss/big bs=1M count=10000 oflag=direct` 吞吐 **> 800 MB/s**
- [ ] 任意低于阈值 CI 红灯
- [ ] writeback cache 一致性测试:两个进程并发写同一文件,内容正确(详见 [risks.md](../risks.md))
- [ ] macOS 路径未受影响(macOS 上 `cargo test` + 集成测试仍全过)

## 关联决策

D14(平台支持矩阵)、D20(双层性能定位)、D21(Linux CI 基准强制)。
